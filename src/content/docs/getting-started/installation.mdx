# Installation

This page covers everything needed to go from zero to a running NestForge application.

<Aside type="note">
  NestForge targets the **stable Rust toolchain**. Nightly is not required.
</Aside>

## Prerequisites

Before you start, make sure you have:

| Requirement         | Notes                                         |
| ------------------- | --------------------------------------------- |
| Stable Rust + Cargo | Install from [rustup.rs](https://rustup.rs)   |
| Git                 | Any recent version                            |
| `protoc`            | Only required if you use the **gRPC** feature |

Check your Rust version with:

```bash
rustc --version
# rustc 1.75.0 (or later)
```

---

## Step 1 â€” Install the CLI

Install the NestForge CLI from crates.io:

```bash
cargo install nestforge-cli
```

Verify the install worked:

```bash
nestforge --version
```

<Aside type="tip">
  If you are working from a local checkout of the NestForge repository instead of crates.io, use:

```bash
cargo install --path crates/nestforge-cli
```

</Aside>

## Step 3 â€” Understand your Cargo.toml

The generated `Cargo.toml` starts with only the core dependency:

```toml
[dependencies]
nestforge = { version = "1", features = [] }
tokio = { version = "1", features = ["full"] }
anyhow = "1"
axum = "0.7"
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

NestForge is **feature-gated**. You only compile and link what you need.

---

## Step 4 â€” Choose your features

Enable features by adding them to your `Cargo.toml`:

```toml
[dependencies]
nestforge = { version = "1", features = ["openapi"] }
```

Here is the full feature reference:

| Feature         | What it adds                                                                   |
| --------------- | ------------------------------------------------------------------------------ |
| `openapi`       | `#[summary]`, `#[tag]`, `#[response]` macros + `/openapi.json` + `/docs` route |
| `config`        | `ConfigModule`, `EnvSchema`, `load_config()` for typed env config              |
| `db`            | `Db`, `connect_db()` for SQL via sqlx                                          |
| `orm`           | `Repo`, `SqlRepo`, `EntityMeta` on top of `db`                                 |
| `graphql`       | `graphql_router`, `GraphQlConfig`, `async_graphql` re-export                   |
| `grpc`          | `NestForgeGrpcFactory`, `GrpcContext`, `prost`/`tonic` re-export               |
| `websockets`    | `websocket_router`, `WebSocketGateway`, `Message` types                        |
| `microservices` | `MicroserviceRegistry`, `InProcessMicroserviceClient`                          |
| `schedule`      | `ScheduleRegistry`, `ScheduledJob`, `start_schedules()`                        |
| `cache`         | `CacheInterceptor`, `CachePolicy`, `cached_response_interceptor`               |
| `mongo`         | `InMemoryMongoRepo`, `MongoConfig`                                             |
| `redis`         | `InMemoryRedisStore`, `RedisConfig`                                            |
| `testing`       | `TestFactory`, `TestingModule` for integration tests                           |
| `data`          | Base data-layer trait; pulled in by `cache`, `mongo`, and `redis`              |

### Common feature combinations

```toml
# REST API with OpenAPI docs
nestforge = { version = "1", features = ["openapi", "config"] }

# REST API with a SQL database
nestforge = { version = "1", features = ["db", "orm", "openapi", "config"] }

# GraphQL API
nestforge = { version = "1", features = ["graphql", "config"] }

# Full-stack transport app
nestforge = { version = "1", features = ["openapi", "graphql", "websockets", "schedule", "config"] }
```

<Aside type="caution">
  Do not enable `grpc` unless you also have `protoc` installed. The gRPC crate
  wraps `tonic` which requires the Protocol Buffers compiler for code generation
  steps.
</Aside>

---

## Step 5 â€” Verify the project builds

```bash
cargo check
```

You should see no errors. If you see `E0433` for unresolved items from `nestforge`, you likely need to add the corresponding feature flag.
