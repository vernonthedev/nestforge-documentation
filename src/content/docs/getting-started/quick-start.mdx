# Quick Start

This guide will walk you through creating a "Hello World" application using the NestForge CLI. You'll learn how to set up the project, define a controller, and run the server.

<Steps>

1. **Create the Project**

   Use the CLI to scaffold a new project. We'll call it `hello-nestforge`.

   ```bash
   nestforge new hello-nestforge
   cd hello-nestforge
   ```

2. **Examine the Structure**

   The CLI generates a modular structure. The key files are:
   - `src/main.rs`: Entry point that bootstraps the application.
   - `src/app_module.rs`: Root module that wires up controllers and providers.

3. **Define a Controller**

   Open `src/main.rs` (or a dedicated controller file if you prefer). By default, the CLI creates a simple controller for you. Let's look at how a basic controller is defined:

   ```rust
   use nestforge::{controller, routes, ApiResult};
   use axum::Json;

   #[controller("/")]
   pub struct AppController;

   #[routes]
   impl AppController {
       #[nestforge::get("/")]
       async fn get_hello() -> ApiResult<String> {
           Ok(Json("Hello from NestForge!".to_string()))
       }
   }
   ```

4. **Register the Controller**

   In `src/app_module.rs`, ensure your `AppController` is registered in the `AppModule`:

   ```rust
   use nestforge::module;
   use crate::AppController;

   #[module(
       controllers = [AppController],
       providers = [],
   )]
   pub struct AppModule;
   ```

5. **Bootstrap the App**

   The `src/main.rs` file bootstraps the application using `NestForgeFactory`:

   ```rust
   use nestforge::NestForgeFactory;
   use crate::app_module::AppModule;

   #[tokio::main]
   async fn main() -> anyhow::Result<()> {
       NestForgeFactory::<AppModule>::create()?
           .listen(3000)
           .await?;
       Ok(())
   }
   ```

6. **Run it!**

   Start your server with Cargo:

   ```bash
   cargo run
   ```

   Visit [http://localhost:3000](http://localhost:3000) in your browser. You should see:
   `"Hello from NestForge!"`

</Steps>

## What's Happening?

- **`#[controller]`**: Defines a base path for all routes within the struct.
- **`#[routes]`**: A required attribute for the `impl` block where your handlers live.
- **`NestForgeFactory`**: The engine that resolves your module graph and mounts the Axum router.
